Auth Flow — Edge Cases Handoff

19 screens across 6 flows. All built with SprouX Design System components + foundation tokens.

Screen Flow
Sign In ──→ Sign Up ──→ Email Verification (OTP) ──→ Success
  │
  ├──→ Forgot Password ──→ Reset Password ──→ Success
  │
  └──→ Success (direct login)
1. Sign In
Default
Default State
Initial load. Email + password fields empty with placeholders.
Trigger
Navigate to /sign-in
Components
Input (x2), Button (default + outline), Checkbox, TextButton, Separator, DecorationInput (eye icon)
Actions
Submit → validate → API call. "Forgot password?" → Forgot flow. "Sign up" → Sign Up flow. "Continue with Google" → OAuth.
Validation Error
Error State
Submit with empty or invalid fields. Inline errors below each field + toast notification (top-right).
Trigger
Click "Sign in" with empty/invalid email or password
Validation Rules
Email: required + valid format (/^[^\s@]+@[^\s@]+\.[^\s@]+$/). Password: required + min 8 chars.
Error Display
Input border → destructive. Text below field → destructive-subtle-foreground (12px). Toast → sonner error variant.
Loading
Loading State
After validation passes, API call in progress. Button disabled + Spinner replaces text.
Trigger
Submit with valid credentials → 1.5s simulated API call
Behavior
Button: disabled + Spinner size="sm" replaces label. All inputs remain enabled (user can still edit). Global error cleared.
Wrong Credentials
Error State
API returns invalid credentials. Alert (destructive) appears above form + toast.
Trigger
Submit with valid email + wrong password (any password except Password1!)
Error Display
Alert variant="destructive" with AlertTitle + AlertDescription + AlertDismiss (x button). Toast error (top-right).
Account Locked
Error State
API returns account locked. Similar to wrong credentials but different message.
Trigger
Submit with email locked@test.com
Error Message
"This account has been locked. Please contact support."
2. Sign Up
Default
Default State
Full name, email, password fields + password strength indicator + terms checkbox.
Components
Input (x3), Button (default + outline), Checkbox, TextButton, Separator, Progress (x4 bars), DecorationInput
All Fields Error
Error State
Submit with all empty fields + unchecked terms. Every field shows error.
Validation Rules
Name: required, min 2 chars. Email: required, valid format. Password: required, min 8 chars. Terms: must be checked.
Weak Password
Interactive State
1 bar filled (score=1). Only length criteria met.
Password Strength Logic
Score 1 (Weak/destructive): length≥8. Score 2 (Fair/warning): +uppercase. Score 3 (Good/primary): +number. Score 4 (Strong/success): +special char.
Strong Password
Success State
All 4 bars filled green (score=4). All criteria met.
Bar Colors
Active: bg-destructive → bg-warning → bg-primary → bg-success. Inactive: bg-muted.
Email Already Taken
Error State
API returns duplicate email. Alert (destructive) above form.
Trigger
Submit with email taken@test.com
Error Message
"An account with this email already exists."
3. Forgot Password
Default
Default State
Simple form: email input + "Send reset link" button + back link.
Trigger
Click "Forgot password?" from Sign In screen
Navigation
"← Back" → Sign In. Submit → Loading → Email Sent.
Email Sent
Success State
Success alert + "Open email app" button. Email field hidden. Shows "Click to resend" link.
Components
Alert variant="success" with AlertTitle + AlertDescription. Button. TextButton ("Click to resend").
Behavior
Title changes to "Check your email". Subtitle shows email address. "Click to resend" resets form.
4. Reset Password
Default
Default State
New password + confirm password fields. Both have eye toggle. Password strength indicator.
Trigger
Click "Open email app" from Forgot Password → Email Sent screen
Password Mismatch
Error State
Confirm password doesn't match new password. Error below confirm field.
Trigger
Submit when new password ≠ confirm password
Error Message
"Passwords do not match"
Success
Success State
Green checkmark icon replaces logo. Form fields removed. Only "Back to sign in" button.
Trigger
Submit with matching passwords after 1.5s API call
Visual
Logo → green circle + CheckCircle2 icon. Title: "Password reset". bg-success + text-success-foreground.
5. Email Verification (OTP)
Default (Empty)
Default State
6-slot OTP input. "Verify email" disabled until 6 digits entered.
Trigger
After successful Sign Up (API returns success)
Components
InputOTP maxLength={6} with InputOTPGroup + InputOTPSlot (x6). Button (disabled). TextButton ("Resend code").
Invalid Code
Error State
Alert (destructive) shows invalid code message.
Trigger
Enter code 000000 and click "Verify email"
Error Message
"Invalid code. Please check and try again."
Expired Code
Error State
Alert (destructive) shows code expired message.
Trigger
Enter code 999999 and click "Verify email"
Resend Behavior
Click "Resend code" → 60s cooldown timer (Resend in {n}s with typo-paragraph-small-medium). Timer text: text-foreground.
6. Success
Welcome
Success State
Green checkmark icon. "You're in!" title. "Go to dashboard" button.
Trigger
After successful Sign In (correct credentials) OR after successful OTP verification
Visual
Logo → bg-success circle + CheckCircle2 icon (size-xl text-success-foreground). "Go to dashboard" → primary button.
